Distribución de elementos / CSS3

display: block

El tipo display: block se usa para colocar elementos uno debajo de otro, no pueden ir dos elementos de tipo bloque en la misma línea

Al usar este atributo, también estamos indicando que el elemento tiene un tamaño definido o bien se lo podemos indicar manualmente al colocar la propiedad de width y height.

Este tipo de elementos se pueden colocar centrados o posicionados a la derecha o izquierda con base a los márgenes de sus contenedores.

Es importante destacar que si se aplica un borde a un elemento de cualquier tipo no importa el atributo de display que contenga, este borde contará como parte del tamaño total, esto quiere decir que si un elemento tiene un ancho 100px con un borde de 2px, el tamaño total sería de 104px.

Para que un elemento se pueda escalar de acuerdo al tamaño de la pantalla es importante declarar un ancho basado en porcentajes, de esta manera al hacer la pantalla más chica el elemento se acomodará de acuerdo al ancho de la pantalla, pero ¿qué pasa si la pantalla es muy grande y no queremos que el elemento ocupe todo el largo de esta? Para eso existe el atributo de max-width, de esta forma se le indica al elemento que puede crecer hasta llegar a un máximo que se le establece, por lo tanto al usar el atributo width con porcentajes y max-width con un tamaño establecido en pixeles, el elemento tendrá un límite a la hora de crecer más no un límite a la hora de hacerse pequeño y puede ocupar el 100% en pantallas más pequeñas.

display: table

El tipo display: table se usa para hacer una distribución de los elementos hijos conforme a una tabla, esto quiere decir que se puede colocar un elemento detrás de otro a manera de filas con varias columnas.

Es importante destacar que un elemento con el atributo table, no puede calcular su tamaño por sí solo, es necesario que se le indique de manera manual este atributo.

Para poder usar este atributo correctamente es necesario que los hijos del elemento tengan un atributo de display en específico para que pueda tener un funcionamiento correcto.

El tipo display: inline-block se usa para colocar elementos dentro de una misma línea, sucede lo mismo que un el atributo table, se le tiene que declarar el tamaño de su contenido manualmente. Puedes lograr lo mismo que con el atributo float, pero aquí el contenedor padre puede calcular su tamaño con base a los hijos inline-block. También este contenedor padre juega un papel importante, porque los elementos inline-block se pueden comportar de distinta manera acorde con el atributo display que contenga el padre.

En la figura 3.1 se pueden ver dos elementos con atributos inline-block y un contenedor padre con el atributo block. Estos elementos se muestran en la misma línea pero hay un pequeño espacio entre ellos, esto es porque el contenedor padre tiene un atributo display: block.

En la figura 3.2 se pueden ver los mismos elementos, pero la diferencia es que ahora no hay un espacio en blanco entre ellos, esto es porque el padre contenedor tiene un atributo display: table. Es aquí que se puede usar el atributo table para el padre, de esta forma los hijos se comportan como si fueran columnas dentro de la tabla. Si los elementos no encajan dentro de la misma línea, estos son colocados por debajo justo como pasa en la figura 3.3.

Otro atributo que puede acompañar a los elementos que contengan el atributo de inline-block, es el de vertical-align: middle | top | bottom. Este atributo sirve para acomodar los elementos con base a su posicionamiento vertical, porque puede haber unos más grandes que los otros como pasa en la figura 3.4, entonces se les puede declarar si se desea que estos se acomoden a partir del margen de arriba con el atributo vertical-align: top, el otro es que se acomoden a manera de que queden centrados verticalmente vertical-align: middle, independientemente del tamaño entre ellos como en la figura 3.5. Por último está el posicionamiento con base al margen de abajo vertical-align: bottom como en la figura 3.6.

Con el atributo inline-block puedes convertir un elemento en dos secciones del mismo tamaño porque se le puede declarar un atributo de tipo width con base en algún porcentaje como 50% tal como se distribuyeron en la figura 3.7. Un error que se puede llegar a cometer es declarar un width: 50% y además agregar un atributo de tipo border. Como se había mencionado en display: block, el borde cuenta a la hora de calcular el tamaño del elemento, pero el problema es que se cuenta como algo extra, no dentro del tamaño declarado en width, por eso en la figura 3.8 los elementos sobrepasaron el 100% del tamaño porque el borde está agregando un extra y hace que el segundo elemento se tire hacia abajo en orden de que puedan entrar los dos elementos en el padre.
